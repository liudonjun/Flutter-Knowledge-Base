# Flutter 包管理详解

> 掌握 Flutter 包管理，高效管理项目依赖。

## 📦 包管理概述

### 1. 包管理工具
```dart
// Flutter 包管理工具
// 1. pub.dev - 官方包仓库
// 2. pubspec.yaml - 项目配置文件
// 3. pubspec.lock - 依赖锁定文件
// 4. flutter pub - 命令行工具

// 包类型
// 1. 依赖包 (dependencies)
// 2. 开发依赖 (dev_dependencies)
// 3. 覆盖依赖 (dependency_overrides)
```

### 2. 包版本管理
```dart
// 版本号格式: major.minor.patch
// 例如: 1.2.3

// 版本约束
// ^1.2.3 - 兼容版本 (>=1.2.3 <2.0.0)
// >=1.2.3 <2.0.0 - 版本范围
// 1.2.3 - 精确版本
// any - 任何版本
// * - 任何版本 (不推荐)
```

## 📝 pubspec.yaml 配置

### 1. 基本配置
```yaml
# 项目基本信息
name: my_flutter_app
description: 我的 Flutter 应用
version: 1.0.0+1
homepage: https://example.com
repository: https://github.com/username/my_flutter_app
issue_tracker: https://github.com/username/my_flutter_app/issues
documentation: https://example.com/docs

# 环境配置
environment:
  sdk: ">=2.17.0 <3.0.0"
  flutter: ">=3.0.0"

# 依赖配置
dependencies:
  flutter:
    sdk: flutter
  
  # UI 组件
  cupertino_icons: ^1.0.2
  
  # 状态管理
  provider: ^6.0.0
  flutter_riverpod: ^2.0.0
  
  # 网络请求
  dio: ^5.0.0
  http: ^1.0.0
  
  # 数据库
  sqflite: ^2.0.0
  hive: ^2.0.0
  
  # 工具库
  shared_preferences: ^2.0.0
  path_provider: ^2.0.0
  intl: ^0.18.0

# 开发依赖
dev_dependencies:
  flutter_test:
    sdk: flutter
  flutter_lints: ^2.0.0
  build_runner: ^2.4.0
  json_serializable: ^6.7.0
  mockito: ^5.0.0

# Flutter 配置
flutter:
  uses-material-design: true
  
  # 资源配置
  assets:
    - assets/images/
    - assets/fonts/
    - assets/data/
  
  # 字体配置
  fonts:
    - family: CustomFont
      fonts:
        - asset: assets/fonts/CustomFont-Regular.ttf
        - asset: assets/fonts/CustomFont-Bold.ttf
          weight: 700
```

### 2. 高级配置
```yaml
# 依赖覆盖 (用于测试或开发)
dependency_overrides:
  # 覆盖特定包的版本
  some_package: ^2.0.0
  
  # 使用本地路径
  local_package:
    path: ../local_package
  
  # 使用 Git 仓库
  git_package:
    git:
      url: https://github.com/user/repo.git
      ref: main
      path: packages/git_package
  
  # 使用 Pub 包
  pub_package:
    hosted:
      name: pub_package
      url: https://pub.example.com

# 执行器配置
executables:
  # 定义可执行文件
  my_app: main
  
# 跳过发布检查
publish_to: none

# 支持的平台
platforms:
  android:
  ios:
  web:
  windows:
  macos:
  linux:
```

## 🔧 包管理命令

### 1. 基本命令
```bash
# 添加依赖
flutter pub add package_name
flutter pub add package_name:^1.0.0
flutter pub add --dev package_name

# 移除依赖
flutter pub remove package_name

# 更新依赖
flutter pub upgrade
flutter pub upgrade --major-versions
flutter pub upgrade package_name

# 获取依赖
flutter pub get

# 检查依赖
flutter pub deps
flutter pub outdated

# 清理缓存
flutter pub cache clean
flutter pub cache repair
```

### 2. 版本管理
```bash
# 查看当前版本
flutter pub deps | grep package_name

# 查看过时依赖
flutter pub outdated

# 更新到最新版本
flutter pub upgrade

# 更新特定包
flutter pub upgrade package_name

# 更新到主版本
flutter pub upgrade --major-versions

# 更新到兼容版本
flutter pub upgrade --no-major-versions
```

### 3. 依赖分析
```bash
# 查看依赖树
flutter pub deps

# 查看依赖图
flutter pub deps --dev

# 查看过时依赖
flutter pub outdated

# 检查依赖冲突
flutter pub deps --no-dev
```

## 📊 依赖类型详解

### 1. dependencies (生产依赖)
```yaml
# 生产依赖 - 应用运行时需要的包
dependencies:
  flutter:
    sdk: flutter
  
  # UI 组件
  cupertino_icons: ^1.0.2
  
  # 状态管理
  provider: ^6.0.0
  
  # 网络请求
  dio: ^5.0.0
  
  # 数据库
  sqflite: ^2.0.0
```

### 2. dev_dependencies (开发依赖)
```yaml
# 开发依赖 - 只在开发时需要的包
dev_dependencies:
  flutter_test:
    sdk: flutter
  
  # 代码分析
  flutter_lints: ^2.0.0
  
  # 代码生成
  build_runner: ^2.4.0
  json_serializable: ^6.7.0
  
  # 测试工具
  mockito: ^5.0.0
  bloc_test: ^9.0.0
  
  # 文档生成
  dartdoc: ^6.0.0
```

### 3. dependency_overrides (依赖覆盖)
```yaml
# 依赖覆盖 - 临时覆盖依赖版本
dependency_overrides:
  # 覆盖包版本
  some_package: ^2.0.0
  
  # 使用本地包
  local_package:
    path: ../local_package
  
  # 使用 Git 包
  git_package:
    git:
      url: https://github.com/user/repo.git
      ref: main
  
  # 使用特定版本
  specific_package: 1.2.3
```

## 🔧 本地包管理

### 1. 本地包引用
```yaml
# 使用本地路径
dependencies:
  local_package:
    path: ../local_package
  
  # 使用相对路径
  sibling_package:
    path: ./packages/sibling_package
  
  # 使用绝对路径
  absolute_package:
    path: /Users/username/packages/absolute_package
```

### 2. 工作区管理
```yaml
# 使用工作区 (monorepo)
# 根目录 pubspec.yaml
name: my_workspace
description: 我的工作区

# 使用 Melos 进行 monorepo 管理
# https://melos.invertase.dev

# 或者使用简单的依赖引用
dependencies:
  package_a:
    path: packages/package_a
  package_b:
    path: packages/package_b
```

### 3. 本地包开发
```bash
# 创建本地包
flutter create --template=package my_local_package

# 引用本地包
# 在主项目的 pubspec.yaml 中添加
dependencies:
  my_local_package:
    path: ../my_local_package

# 开发本地包
cd my_local_package
flutter pub get
flutter test

# 发布本地包
flutter pub publish
```

## 🌐 Git 包管理

### 1. Git 依赖引用
```yaml
# 使用 Git 仓库
dependencies:
  # 基本引用
  git_package:
    git: https://github.com/user/repo.git
  
  # 指定分支
  git_package_branch:
    git:
      url: https://github.com/user/repo.git
      ref: main
  
  # 指定标签
  git_package_tag:
    git:
      url: https://github.com/user/repo.git
      ref: v1.0.0
  
  # 指定提交
  git_package_commit:
    git:
      url: https://github.com/user/repo.git
      ref: abc123def456
  
  # 指定子目录
  git_package_path:
    git:
      url: https://github.com/user/repo.git
      ref: main
      path: packages/sub_package
```

### 2. Git 包管理最佳实践
```yaml
# 最佳实践
dependencies:
  # 使用标签而不是分支
  git_package:
    git:
      url: https://github.com/user/repo.git
      ref: v1.0.0  # 使用标签
  
  # 避免使用 main 分支
  # git_package:
  #   git:
  #     url: https://github.com/user/repo.git
  #     ref: main  # 不推荐
  
  # 使用提交哈希确保稳定性
  git_package:
    git:
      url: https://github.com/user/repo.git
      ref: abc123def456  # 提交哈希
```
