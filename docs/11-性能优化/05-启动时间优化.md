# 应用启动时间优化

> 掌握 Flutter 应用启动时间优化技巧。

## 📖 启动基础

### 1. 启动类型

```dart
// 启动类型
class StartupTypes {
  void explain() {
    print('''
    启动类型：
    
    // 1. 冷启动
    // - 应用完全关闭后启动
    // - 需要加载所有资源
    // - 耗时最长
    
    // 2. 热启动
    // - 应用在后台，重新启动
    // - 部分资源已加载
    // - 耗时较短
    
    // 3. 温启动
    // - 应用被系统杀死后启动
    // - 需要重新加载资源
    // - 耗时中等
    
    // 4. 启动时间组成
    // - 初始化时间：Flutter 引擎初始化
    // - 加载时间：资源加载
    // - 渲染时间：首帧渲染
    // - 交互时间：用户可交互
    ''');
  }
}
```

### 2. 启动时间测量

```dart
// 启动时间测量
class StartupTimeMeasurement {
  void explain() {
    print('''
    启动时间测量：
    
    // 1. 使用 WidgetsBinding
    void main() {
      final binding = WidgetsFlutterBinding.ensureInitialized();
      
      binding.addPostFrameCallback((_) {
        final startupTime = DateTime.now().difference(startTime);
        print('启动时间: ${startupTime.inMilliseconds}ms');
      });
      
      runApp(MyApp());
    }
    
    // 2. 使用 Performance Overlay
    MaterialApp(
      showPerformanceOverlay: true,
      home: MyHomePage(),
    )
    
    // 3. 使用 DevTools Timeline
    // - 打开 DevTools
    // - 切换到 Performance 面板
    // - 开始录制
    // - 启动应用
    // - 分析启动时间
    
    // 4. 自定义启动时间测量
    class StartupTracker {
      final DateTime _startTime = DateTime.now();
      final List<StartupEvent> _events = [];
      
      void recordEvent(String name) {
        final event = StartupEvent(
          name: name,
          timestamp: DateTime.now(),
          duration: DateTime.now().difference(_startTime),
        );
        _events.add(event);
      }
      
      void printReport() {
        print('启动时间报告:');
        for (final event in _events) {
          print('  ${event.name}: ${event.duration.inMilliseconds}ms');
        }
      }
    }
    ''');
  }
}
```

### 1. 减少初始化时间

// 减少初始化时间
class ReduceInitTime {
  void explain() {
    print('''
    减少初始化时间：

    class App {
      late final Database _database;

        // 延迟初始化
        _database = await Database.init();
        _apiService = await ApiService.init();
      }
    }
    
    // 2. 使用 Isolate
    Future<void> heavyTask() async {
      final result = await compute(heavyFunction, data);
      return result;
    }
    
    // 3. 减少插件数量
    // - 移除不必要的插件
    // - 合并功能相似的插件
    // - 使用轻量级插件
    
    // 4. 优化依赖
    // - 减少第三方库
    // - 使用 tree-shaking
    // - 按需加载库
    
    // 5. 使用 Flutter 初始化配置
    void main() {
      // 只初始化必要的服务
      WidgetsFlutterBinding.ensureInitialized();
      
      // 延迟初始化其他服务

        // 初始化其他服务

    ''');

class OptimizeResourceLoading {

      'assets/image.jpg',
      width: 100,
      height: 100,

        return CircularProgressIndicator();

## ⚠️ 注意事项

1. **冷启动**：冷启动时间应小于 2 秒
2. **热启动**：热启动时间应小于 1 秒
3. **首屏加载**：首屏内容应尽快显示
4. **用户交互**：尽快让用户可以交互
5. **测试设备**：在低端设备上测试启动时间
