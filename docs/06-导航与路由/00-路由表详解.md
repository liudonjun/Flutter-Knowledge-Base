# 路由表详解

> 深入理解 Flutter 中的路由表。

## 📖 路由表基础

### 1. 什么是路由表

```dart
// 路由表概念
class RouteTableConcept {
  void explain() {
    print('''
    路由表概念：
    
    // 1. 什么是路由表
    // - 路由名称到页面的映射
    // - 集中管理路由配置
    // - 支持命名路由
    // - 便于路由管理
    
    // 2. 路由表的特点
    // - 集中管理：所有路由在一处定义
    // - 类型安全：编译时检查路由名称
    // - 易于维护：修改路由配置简单
    // - 支持参数：可以传递路由参数
    
    // 3. 路由表的使用场景
    // - 大型应用：需要管理大量路由
    // - 团队协作：统一路由规范
    // - 深度链接：支持外部链接
    // - 路由分析：便于路由统计
    ''');
  }
}
```

### 2. 路由表配置

```dart
// 路由表配置
class RouteTableConfiguration {
  void explain() {
    print('''
    路由表配置：
    
    // 1. 定义路由名称
    class AppRoutes {
      static const String HOME = '/';
      static const String DETAILS = '/details';
      static const String SETTINGS = '/settings';
      static const String PROFILE = '/profile';
    }
    
    // 2. 配置路由表
    MaterialApp(
      initialRoute: AppRoutes.HOME,
      routes: {
        AppRoutes.HOME: (context) => HomePage(),
        AppRoutes.DETAILS: (context) => DetailsPage(),
        AppRoutes.SETTINGS: (context) => SettingsPage(),
        AppRoutes.PROFILE: (context) => ProfilePage(),
      },
    )
    
    // 3. 使用 onGenerateRoute
    MaterialApp(
      onGenerateRoute: (settings) {
        switch (settings.name) {
          case AppRoutes.HOME:
            return MaterialPageRoute(
              builder: (context) => HomePage(),
            );
          case AppRoutes.DETAILS:
            final args = settings.arguments as Map<String, dynamic>;
            return MaterialPageRoute(
              builder: (context) => DetailsPage(id: args['id']),
            );
          default:
            return MaterialPageRoute(
              builder: (context) => NotFoundPage(),
            );
        }
      },
    )
    ''');
  }
}
```

## 🔧 路由表使用

### 1. 命名路由

```dart
// 命名路由
class NamedRoutes {
  void explain() {
    print('''
    命名路由：
    
    // 1. 定义命名路由
    class AppRoutes {
      static const String HOME = '/';
      static const String DETAILS = '/details';
      static const String SETTINGS = '/settings';
    }
    
    // 2. 使用命名路由
    Navigator.pushNamed(context, AppRoutes.DETAILS);
    
    // 3. 带参数导航
    Navigator.pushNamed(
      context,
      AppRoutes.DETAILS,
      arguments: {'id': 42},
    );
    
    // 4. 获取路由参数
    final args = ModalRoute.of(context)!.settings.arguments as Map<String, dynamic>;
    final id = args['id'];
    ''');
  }
}
```

### 2. 动态路由表

```dart
// 动态路由表
class DynamicRouteTable {
  void explain() {
    print('''
    动态路由表：
    
    // 1. 动态路由表
    class DynamicRouteTable {
      static Map<String, WidgetBuilder> _routes = {};
      
      static void addRoute(String name, WidgetBuilder builder) {
        _routes[name] = builder;
      }
      
      static void removeRoute(String name) {
        _routes.remove(name);
      }
      
      static Map<String, WidgetBuilder> get routes => _routes;
    }
    
    // 2. 使用动态路由表
    MaterialApp(
      routes: DynamicRouteTable.routes,
    )
    
    // 3. 动态添加路由
    void addDynamicRoutes() {
      DynamicRouteTable.addRoute('/product', (context) => ProductPage());
      DynamicRouteTable.addRoute('/user', (context) => UserPage());
    }
    
    // 4. 动态路由生成
    MaterialApp(
      onGenerateRoute: (settings) {
        final routeBuilder = DynamicRouteTable.routes[settings.name];
        if (routeBuilder != null) {
          return MaterialPageRoute(builder: routeBuilder);
        }
        return MaterialPageRoute(builder: (context) => NotFoundPage());
      },
    )
    ''');
  }
}
```

## ⚠️ 注意事项

1. **路由表应该统一管理**
2. **注意路由名称唯一性**
3. **处理未定义的路由**
4. **考虑路由性能**
