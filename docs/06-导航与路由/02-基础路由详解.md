// 导航操作
class NavigationOperations {
  /*
  导航操作：
  
  1. push 操作
     - Navigator.push：压入新页面
     - Navigator.pushNamed：使用命名路由
     - Navigator.pushReplacement：替换当前页面
  
  2. pop 操作
     - Navigator.pop：弹出当前页面
     - Navigator.popUntil：弹出直到某个页面
     - Navigator.canPop：检查是否可以弹出
  
  3. 高级操作
     - Navigator.pushAndRemoveUntil：压入并移除
     - Navigator.removeRoute：移除路由
     - Navigator.replace：替换路由
  */
  
  void explain() {
    print('''
    导航操作：
    
    1. push 操作
       Navigator.push：
       - 压入新页面
       - 返回 Future
       - 可以获取返回值
       
       Navigator.pushNamed：
       - 使用命名路由
       - 需要路由表
       - 支持参数传递
       
       Navigator.pushReplacement：
       - 替换当前页面
       - 不会返回原页面
       - 用于登录后跳转
    
    2. pop 操作
       Navigator.pop：
       - 弹出当前页面
       - 可以传递返回值
       - 关闭对话框
       
       Navigator.popUntil：
       - 弹出直到某个页面
       - 使用谓词函数
       - 用于返回首页
       
       Navigator.canPop：
       - 检查是否可以弹出
       - 防止空栈弹出
    
    3. 高级操作
       Navigator.pushAndRemoveUntil：
       - 压入并移除
       - 清理路由栈
       - 用于登录后跳转
       
       Navigator.removeRoute：
       - 移除指定路由
       - 从栈中移除
       
       Navigator.replace：
       - 替换路由
       - 替换指定路由
    
    示例：
    // push
    Navigator.push(context, MaterialPageRoute(builder: (context) => NextPage()));
    
    // pushNamed
    Navigator.pushNamed(context, '/next');
    
    // pop
    Navigator.pop(context);
    
    // popUntil
    Navigator.popUntil(context, ModalRoute.withName('/'));
    
    // pushAndRemoveUntil
    Navigator.pushAndRemoveUntil(
      context,
      MaterialPageRoute(builder: (context) => HomePage()),
      (route) => false,
    );
    ''');
  }
}

       - 类型安全
       - 简单直接
       - 编译时检查

    );

## 📖 总结

### 路由管理核心概念

| 概念 | 描述 | 使用场景 |
|------|------|----------|
| **MaterialPageRoute** | 页面路由 | 基础页面跳转 |
| **Navigator.push** | 压入页面 | 页面导航 |
| **Navigator.pop** | 弹出页面 | 返回上一页 |
| **命名路由** | 使用名称跳转 | 复杂路由管理 |
| **路由参数** | 参数传递 | 页面间数据传递 |
| **路由守卫** | 路由拦截 | 权限验证、登录检查 |
