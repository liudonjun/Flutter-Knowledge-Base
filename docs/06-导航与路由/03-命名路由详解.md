// 命名路由导航
class NamedRouteNavigation {
  /*
  命名路由导航：
  
  1. 基本导航
     - Navigator.pushNamed：压入命名路由
     - Navigator.pushReplacementNamed：替换当前路由
     - Navigator.pushNamedAndRemoveUntil：压入并移除
  
  2. 参数传递
     - 通过 arguments 传递
     - 在页面中获取参数
     - 支持复杂参数
  
  3. 返回值
     - 使用 Future 获取返回值
     - 在页面中返回值
  */
  
  void explain() {
    print('''
    命名路由导航：
    
    1. 基本导航
       Navigator.pushNamed：
       - 压入命名路由
       - 返回 Future
       - 可以获取返回值
       
       Navigator.pushReplacementNamed：
       - 替换当前路由
       - 不会返回原页面
       - 用于登录后跳转
       
       Navigator.pushNamedAndRemoveUntil：
       - 压入并移除
       - 清理路由栈
       - 用于返回首页
    
    2. 参数传递
       - 通过 arguments 传递
       - 在页面中获取参数
       - 支持复杂参数
       - 类型安全
    
    3. 返回值
       - 使用 Future 获取返回值
       - 在页面中返回值
       - 支持异步操作
    
    示例：
    // 基本导航
    Navigator.pushNamed(context, '/detail');
    
    // 带参数导航
    Navigator.pushNamed(
      context,
      '/detail',
      arguments: {'id': 1, 'title': '标题'},
    );
    
    // 获取返回值
    final result = await Navigator.pushNamed(context, '/result');
    
    // 替换当前路由
    Navigator.pushReplacementNamed(context, '/home');
    
    // 压入并移除
    Navigator.pushNamedAndRemoveUntil(
      context,
      '/home',
      (route) => false,
    );
    ''');
  }
}

          if (id == null) {
            return MaterialPageRoute(
              builder: (context) => ErrorPage(
                message: '缺少用户ID',
              ),
            );
          }
          
          return MaterialPageRoute(
            builder: (context) => UserPage(userId: id),
          );
        }
        
        return null;
      },
    )
    ''');
  }
}

## 📖 总结

### 命名路由核心概念

| 概念 | 描述 | 使用场景 |
|------|------|----------|
| **路由表** | 预定义的路由映射 | 集中管理路由 |
| **命名路由** | 使用名称进行导航 | 复杂路由管理 |
| **路由参数** | 通过路由传递参数 | 页面间数据传递 |
| **动态路由** | 动态生成路由 | 复杂路由逻辑 |
| **路由守卫** | 路由拦截和验证 | 权限控制、登录检查 |

### 最佳实践总结

1. **路由管理**：使用命名路由集中管理路由

3. **参数验证**：进行参数验证和类型检查
4. **错误处理**：处理错误情况和未知路由
5. **性能优化**：避免路由表过于复杂
