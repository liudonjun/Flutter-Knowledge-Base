# 路由守卫详解

> 深入理解 Flutter 中的路由守卫机制。

## 📖 路由守卫基础

### 1. 什么是路由守卫

```dart
// 路由守卫概念
class RouteGuardConcept {
  void explain() {
    print('''
    路由守卫概念：
    
    // 1. 什么是路由守卫
    // - 路由访问控制机制
    // - 在路由跳转前后执行逻辑
    // - 用于权限验证、日志记录等
    // - 保护特定路由
    
    // 2. 路由守卫的特点
    // - 拦截导航：可以阻止或重定向
    // - 权限控制：验证用户权限
    // - 日志记录：记录导航日志
    // - 数据预加载：在导航前加载数据
    
    // 3. 路由守卫的使用场景
    // - 登录验证：未登录重定向到登录页
    // - 权限检查：无权限访问重定向
    // - 数据验证：验证路由参数
    // - 日志追踪：记录用户导航路径
    ''');
  }
}
```

### 2. 路由守卫实现

```dart
// 路由守卫实现
class RouteGuardImplementation {
  void explain() {
    print('''
    路由守卫实现：
    
    // 1. 使用 Navigator 的 onGenerateRoute
    MaterialApp(
      onGenerateRoute: (settings) {
        // 检查权限
        if (settings.name == '/admin' && !isAdmin) {
          return MaterialPageRoute(
            builder: (context) => AccessDeniedPage(),
          );
        }
        return MaterialPageRoute(
          builder: (context) => getPage(settings.name),
        );
      },
    )
    
    // 2. 使用路由包装器
    class ProtectedRoute extends StatelessWidget {
      final Widget child;
      final bool Function() checkPermission;
      
      const ProtectedRoute({
        required this.child,
        required this.checkPermission,
      });
      
      @override
      Widget build(BuildContext context) {
        if (checkPermission()) {
          return child;
        } else {
          return AccessDeniedPage();
        }
      }
    }
    
    // 3. 使用 MaterialPageRoute 的 builder
    MaterialPageRoute(
      builder: (context) {
        if (!isLoggedIn) {
          return LoginPage();
        }
        return AdminPage();
      },
    )
    ''');
  }
}
```

## 🔧 路由守卫模式

### 1. 登录守卫

```dart
// 登录守卫
class LoginGuard {
  void explain() {
    print('''
    登录守卫：
    
    // 1. 检查登录状态
    class AuthGuard {
      static bool isLoggedIn = false;
      
      static Widget checkAuth(Widget page) {
        if (isLoggedIn) {
          return page;
        } else {
          return LoginPage();
        }
      }
    }
    
    // 2. 使用登录守卫
    MaterialPageRoute(
      builder: (context) => AuthGuard.checkAuth(ProfilePage()),
    )
    
    // 3. 登录后重定向
    class LoginPage extends StatelessWidget {
      @override
      Widget build(BuildContext context) {
        return ElevatedButton(
          onPressed: () {
            AuthGuard.isLoggedIn = true;
            Navigator.pushReplacementNamed(context, '/profile');
          },
          child: Text('Login'),
        );
      }
    }
    
    // 4. 登出处理
    void logout() {
      AuthGuard.isLoggedIn = false;
      Navigator.pushNamedAndRemoveUntil(
        context,
        '/login',
        (route) => false,
      );
    }
    ''');
  }
}
```

### 2. 权限守卫

```dart
// 权限守卫
class PermissionGuard {
  void explain() {
    print('''
    权限守卫：
    
    // 1. 权限检查
    class PermissionGuard {
      static bool hasPermission(String permission) {
        // 检查用户权限
        return userPermissions.contains(permission);
      }
      
      static Widget checkPermission(
        String permission,
        Widget page,
      ) {
        if (hasPermission(permission)) {
          return page;
        } else {
          return AccessDeniedPage();
        }
      }
    }
    
    // 2. 使用权限守卫
    MaterialPageRoute(
      builder: (context) => PermissionGuard.checkPermission(
        'admin',
        AdminPage(),
      ),
    )
    
    // 3. 多权限检查
    class MultiPermissionGuard {
      static Widget checkPermissions(
        List<String> permissions,
        Widget page,
      ) {
        for (final permission in permissions) {
          if (!PermissionGuard.hasPermission(permission)) {
            return AccessDeniedPage();
          }
        }
        return page;
      }
    }
    
    // 4. 角色守卫
    class RoleGuard {
      static bool hasRole(String role) {
        return userRoles.contains(role);
      }
      
      static Widget checkRole(String role, Widget page) {
        if (hasRole(role)) {
          return page;
        } else {
          return AccessDeniedPage();
        }
      }
    }
    ''');
  }
}
```

## ⚠️ 注意事项

1. **路由守卫应该简洁高效**
2. **避免在守卫中执行耗时操作**
3. **考虑用户体验**
4. **处理守卫失败的情况**
