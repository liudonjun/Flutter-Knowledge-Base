# 命名路由详解

> 深入理解 Flutter 中的命名路由。

## 📖 命名路由基础

### 1. 什么是命名路由

```dart
// 命名路由概念
class NamedRoutesConcept {
  void explain() {
    print('''
    命名路由概念：
    
    // 1. 什么是命名路由
    // - 使用字符串名称标识路由
    // - 集中管理路由配置
    // - 支持参数传递
    // - 便于路由管理
    
    // 2. 命名路由的特点
    // - 集中管理：所有路由在一处定义
    // - 类型安全：编译时检查路由名称
    // - 易于维护：修改路由配置简单
    // - 支持参数：可以传递路由参数
    
    // 3. 命名路由的使用场景
    // - 大型应用：需要管理大量路由
    // - 团队协作：统一路由规范
    // - 深度链接：支持外部链接
    // - 路由分析：便于路由统计
    ''');
  }
}
```

### 2. 命名路由配置

```dart
// 命名路由配置
class NamedRoutesConfiguration {
  void explain() {
    print('''
    命名路由配置：
    
    // 1. 定义路由名称
    class AppRoutes {
      static const String HOME = '/';
      static const String DETAILS = '/details';
      static const String SETTINGS = '/settings';
      static const String PROFILE = '/profile';
    }
    
    // 2. 配置路由表
    MaterialApp(
      initialRoute: AppRoutes.HOME,
      routes: {
        AppRoutes.HOME: (context) => HomePage(),
        AppRoutes.DETAILS: (context) => DetailsPage(),
        AppRoutes.SETTINGS: (context) => SettingsPage(),
        AppRoutes.PROFILE: (context) => ProfilePage(),
      },
    )
    
    // 3. 使用 onGenerateRoute
    MaterialApp(
      onGenerateRoute: (settings) {
        switch (settings.name) {
          case AppRoutes.HOME:
            return MaterialPageRoute(
              builder: (context) => HomePage(),
            );
          case AppRoutes.DETAILS:
            final args = settings.arguments as Map<String, dynamic>;
            return MaterialPageRoute(
              builder: (context) => DetailsPage(id: args['id']),
            );
          default:
            return MaterialPageRoute(
              builder: (context) => NotFoundPage(),
            );
        }
      },
    )
    ''');
  }
}
```

## 🔧 命名路由使用

### 1. 基本导航

```dart
// 基本导航
class BasicNavigation {
  void explain() {
    print('''
    基本导航：
    
    // 1. 导航到命名路由
    Navigator.pushNamed(context, AppRoutes.DETAILS);
    
    // 2. 带参数导航
    Navigator.pushNamed(
      context,
      AppRoutes.DETAILS,
      arguments: {'id': 42},
    );
    
    // 3. 返回上一页
    Navigator.pop(context);
    
    // 4. 替换当前页面
    Navigator.pushReplacementNamed(context, AppRoutes.SETTINGS);
    
    // 5. 清空导航栈
    Navigator.pushNamedAndRemoveUntil(
      context,
      AppRoutes.HOME,
      (route) => false,
    );
    
    // 6. 获取路由参数
    final args = ModalRoute.of(context)!.settings.arguments as Map<String, dynamic>;
    final id = args['id'];
    ''');
  }
}
```

### 2. 动态路由

```dart
// 动态路由
class DynamicRoutes {
  void explain() {
    print('''
    动态路由：
    
    // 1. 路径参数
    // /user/:id
    onGenerateRoute: (settings) {
      final uri = Uri.parse(settings.name!);
      
      if (uri.pathSegments.length == 2 && uri.pathSegments[0] == 'user') {
        final id = uri.pathSegments[1];
        return MaterialPageRoute(
          builder: (context) => UserPage(id: id),
        );
      }
      
      return MaterialPageRoute(
        builder: (context) => NotFoundPage(),
      );
    }
    
    // 2. 查询参数
    // /search?query=flutter
    onGenerateRoute: (settings) {
      final uri = Uri.parse(settings.name!);
      
      if (uri.path == '/search') {
        final query = uri.queryParameters['query'];
        return MaterialPageRoute(
          builder: (context) => SearchPage(query: query),
        );
      }
      
      return MaterialPageRoute(
        builder: (context) => NotFoundPage(),
      );
    }
    
    // 3. 路由守卫
    onGenerateRoute: (settings) {
      if (settings.name == AppRoutes.PROFILE) {
        if (!isLoggedIn) {
          return MaterialPageRoute(
            builder: (context) => LoginPage(),
          );
        }
      }
      
      return MaterialPageRoute(
        builder: (context) => getPage(settings.name),
      );
    }
    ''');
  }
}
```

## ⚠️ 注意事项

1. **命名路由应该统一管理**
2. **注意路由参数类型安全**
3. **处理未定义的路由**
4. **考虑路由性能**
