# 状态更新详解

> 深入理解 Flutter 中的状态更新机制。

## 📖 状态更新基础

### 1. 什么是状态更新

```dart
// 状态更新概念
class StateUpdateConcept {
  void explain() {
    print('''
    状态更新概念：
    
    // 1. 什么是状态更新
    // - 修改应用状态
    // - 触发 UI 重建
    // - 保持状态一致
    // - 响应用户交互
    
    // 2. 状态更新的特点
    // - 不可变性：状态应该是不可变的
    // - 响应式：状态变化触发 UI 更新
    // - 可预测：状态更新应该是可预测的
    // - 可测试：状态更新应该是可测试的
    
    // 3. 状态更新的方式
    // - setState：局部状态更新
    // - notifyListeners：通知监听者
    // - emit：BLoC 状态更新
    // - dispatch：Redux 状态更新
    
    // 4. 状态更新的最佳实践
    // - 使用不可变状态
    // - 避免直接修改状态
    // - 使用纯函数
    // - 处理异步更新
    ''');
  }
}
```

### 2. 状态更新实现

```dart
// 状态更新实现
class StateUpdateImplementation {
  void explain() {
    print('''
    状态更新实现：
    
    // 1. setState 更新
    void updateWithSetState() {
      setState(() {
        _count++;
      });
    }
    
    // 2. notifyListeners 更新
    void updateWithNotifyListeners() {
      _count++;
      notifyListeners();
    }
    
    // 3. BLoC 更新
    void updateWithBLoC() {
      emit(CounterState(_count + 1));
    }
    
    // 4. Redux 更新
    void updateWithRedux() {
      store.dispatch(IncrementAction());
    }
    
    // 5. Riverpod 更新
    void updateWithRiverpod() {
      ref.read(counterProvider.notifier).state++;
    }
    
    // 6. GetX 更新
    void updateWithGetX() {
      count.value++;
    }
    ''');
  }
}
```
