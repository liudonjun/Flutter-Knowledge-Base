# 状态粒度详解

> 深入理解 Flutter 中的状态粒度概念。

## 📖 状态粒度基础

### 1. 什么是状态粒度

```dart
// 状态粒度概念
class StateGranularityConcept {
  void explain() {
    print('''
    状态粒度概念：
    
    // 1. 什么是状态粒度
    // - 状态的细分程度
    // - 状态的组织方式
    // - 状态的管理范围
    // - 状态的更新范围
    
    // 2. 状态粒度的特点
    // - 细粒度：状态细分程度高
    // - 粗粒度：状态细分程度低
    // - 影响性能：粒度影响重建范围
    // - 影响复杂度：粒度影响代码复杂度
    
    // 3. 状态粒度的选择
    // - 细粒度：性能要求高
    // - 粗粒度：开发效率高
    // - 根据需求选择
    // - 平衡性能和复杂度
    
    // 4. 状态粒度的使用场景
    // - 表单：细粒度状态
    // - 列表：粗粒度状态
    // - 配置：细粒度状态
    // - 用户信息：粗粒度状态
    ''');
  }
}
```

### 2. 状态粒度实现

```dart
// 状态粒度实现
class StateGranularityImplementation {
  void explain() {
    print('''
    状态粒度实现：
    
    // 1. 细粒度状态
    class FormState {
      final String name;
      final String email;
      final String password;
      
      const FormState({
        required this.name,
        required this.email,
        required this.password,
      });
    }
    
    // 2. 粗粒度状态
    class AppState {
      final User user;
      final Settings settings;
      final List<Item> items;
      
      const AppState({
        required this.user,
        required this.settings,
        required this.items,
      });
    }
    
    // 3. 使用 Selector 优化粒度
    Selector<FormState, String>(
      selector: (context, state) => state.name,
      builder: (context, name, child) {
        return TextField(
          controller: TextEditingController(text: name),
        );
      },
    )
    
    // 4. 粒度最佳实践
    // - 根据更新频率选择粒度
    // - 根据性能要求选择粒度
    // - 根据代码复杂度选择粒度
    // - 平衡性能和复杂度
    ''');
  }
}
```
