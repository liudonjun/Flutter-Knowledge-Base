# 状态监听详解

> 深入理解 Flutter 中的状态监听机制。

## 📖 状态监听基础

### 1. 什么是状态监听

```dart
// 状态监听概念
class StateListeningConcept {
  void explain() {
    print('''
    状态监听概念：
    
    // 1. 什么是状态监听
    // - 监听状态变化
    // - 响应状态更新
    // - 触发 UI 重建
    // - 实现响应式
    
    // 2. 状态监听的特点
    // - 响应式：状态变化自动响应
    // - 自动化：无需手动检查
    // - 高效：只响应需要的变化
    // - 灵活：支持多种监听方式
    
    // 3. 状态监听的方式
    // - Provider：自动监听
    // - BLoC：Stream 监听
    // - Redux：Store 监听
    // - GetX：Obx 监听
    
    // 4. 状态监听的使用场景
    // - UI 更新
    // - 数据同步
    // - 事件处理
    // - 状态变化
    ''');
  }
}
```

### 2. 状态监听实现

```dart
// 状态监听实现
class StateListeningImplementation {
  void explain() {
    print('''
    状态监听实现：
    
    // 1. Provider 监听
    Consumer<Counter>(
      builder: (context, counter, child) {
        return Text('Count: ${counter.count}');
      },
    )
    
    // 2. BLoC 监听
    BlocBuilder<CounterBloc, CounterState>(
      builder: (context, state) {
        return Text('Count: ${state.count}');
      },
    )
    
    // 3. Redux 监听
    StoreConnector<AppState, int>(
      converter: (store) => store.state.count,
      builder: (context, count) {
        return Text('Count: $count');
      },
    )
    
    // 4. GetX 监听
    Obx(() => Text('Count: ${controller.count}'))
    
    // 5. Stream 监听
    StreamBuilder<int>(
      stream: counterStream,
      builder: (context, snapshot) {
        return Text('Count: ${snapshot.data}');
      },
    )
    ''');
  }
}
```
