# 用户系统设计

> Flutter 应用中账号、鉴权、会话与第三方登录的模块划分与实现要点。

## 模块划分

| 子模块 | 职责 |
| --- | --- |
| **注册登录** | 邮箱/手机、验证码、密码；OAuth 入口 |
| **会话** | Access/Refresh Token、自动刷新、登出 |
| **资料** | 昵称、头像、绑定手机；隐私字段 |
| **权限** | 角色、功能开关（与后端 RBAC 对齐） |
| **安全** | 设备管理、敏感操作二次验证 |

## 推荐架构

```
features/auth/
├── presentation/     # 登录页、绑定页
├── domain/           # AuthRepository 接口、User 实体
└── data/             # API、TokenStore、OAuth 适配器
core/
├── network/          # AuthInterceptor、401 刷新
└── storage/          # [[20-secure_storage详解]]
```

Presentation 不直接调 Google/微信 SDK；通过 `OAuthProvider` 接口注入，便于测试与换渠道。

## 会话与 Token

```dart
abstract class TokenStore {
  Future<String?> get accessToken;
  Future<void save({required String access, String? refresh});
  Future<void clear();
}

class AuthInterceptor extends Interceptor {
  @override
  void onRequest(RequestOptions options, RequestInterceptorHandler handler) async {
    final token = await tokenStore.accessToken;
    if (token != null) {
      options.headers['Authorization'] = 'Bearer $token';
    }
    handler.next(options);
  }

  @override
  void onError(DioException err, ErrorInterceptorHandler handler) async {
    if (err.response?.statusCode == 401) {
      final refreshed = await authRepo.tryRefreshToken();
      if (refreshed) {
        final clone = await dio.fetch(err.requestOptions);
        return handler.resolve(clone);
      }
      await authRepo.logout();
    }
    handler.next(err);
  }
}
```

- Access Token 短有效期；Refresh Token 存 secure storage
- 登出：清本地 Token + 可选 revoke 服务端会话

## 第三方登录矩阵

| 渠道 | 文档 | 客户端产出 | 服务端 |
| --- | --- | --- | --- |
| 微信 | [[38-wechat_login详解]] | `code` | code → access_token → openid/unionid |
| Google | [[36-google_sign_in详解]] | `idToken` | 验签 JWT，`sub` 绑定 |
| Apple | [[37-apple_sign_in详解]] | `identityToken` | 验 Apple JWT，`sub` 绑定 |
| Facebook | [[39-facebook_login详解]] | `accessToken` | debug_token + `/me` |

统一后端接口形态示例：

```text
POST /auth/oauth/{provider}  body: { token | code, ... }
→ { access_token, refresh_token, user }
```

首次 OAuth 可返回 `need_bind_phone: true`，引导补全手机号。

## 状态管理

- **Riverpod**：`AuthNotifier` 暴露 `AsyncValue<User?>`；路由 redirect 监听登录态
- **Bloc**：`AuthBloc` 事件 `AppStarted`、`LoggedIn`、`LoggedOut`
- 全局只需 **一份** 当前用户；资料页用 `family` 或单独 `ProfileRepository`

## 路由守卫

```dart
redirect: (context, state) {
  final loggedIn = authState.isLoggedIn;
  final loggingIn = state.matchedLocation == '/login';
  if (!loggedIn && !loggingIn) return '/login';
  if (loggedIn && loggingIn) return '/';
  return null;
},
```

Deep Link 进支付/聊天页时，未登录先带 `redirect` 参数跳登录。

## 安全与合规

1. 密码不明文存本地；OAuth 不存 AppSecret
2. 头像/昵称缓存与账号解绑时清除
3. 隐私政策、账号注销入口（尤其 iOS + GDPR）
4. 敏感操作（改密、绑卡）需 re-auth 或短信验证

## 测试建议

| 用例 | 说明 |
| --- | --- |
| 登录成功/失败 | Mock `AuthRepository` |
| Token 过期刷新 | 拦截器 401 → 重试一次 |
| 登出清态 | Provider/Bloc 与路由回到登录页 |
| OAuth 取消 | 各 SDK 返回 cancel 不 crash |
