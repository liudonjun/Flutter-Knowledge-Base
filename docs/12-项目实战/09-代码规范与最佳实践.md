## 📖 代码规范

### 1. 命名规范

```dart
// 命名规范
class NamingConventions {
  /*
  命名规范：
  
  1. 类名
     - 大驼峰：MyClass
     - Widget：XxxWidget, XxxPage, XxxScreen
     - 抽象类：AbstractXxx 或 Xxx
  
  2. 变量和方法
     - 小驼峰：myVariable, myMethod
     - 私有：_myPrivateVariable
     - 常量：myConstant 或 MY_CONSTANT
  
  3. 文件名
     - 小写蛇形：my_file_name.dart
     - 与主要类名对应
  */
  
  void explain() {
    print('''
    命名规范：
    
    1. 类名 - 大驼峰 (PascalCase)
       // Widget
       class MyWidget extends StatelessWidget { }
       class UserProfilePage extends StatefulWidget { }
       class CustomButton extends StatelessWidget { }
       
       // Model
       class User { }
       class UserProfile { }
       class ApiResponse { }
       
       // 抽象类
       abstract class Repository { }
       abstract class BaseRepository implements Repository { }
    
    2. 变量和方法 - 小驼峰 (camelCase)
       // 变量
       String userName = '';
       int userAge = 0;
       bool isLoading = false;
       
       // 私有变量
       String _privateVariable = '';
       final List<Item> _items = [];
       
       // 方法
       void fetchData() { }
       Future<User> getUserById(int id) { }
       bool isValidEmail(String email) { }
       
       // 回调
       void Function()? onTap;
       ValueChanged<String>? onChanged;
    
    3. 常量
       // 编译时常量
       const String apiBaseUrl = 'https://api.example.com';
       const int maxRetries = 3;
       
       // 全局常量
       const double defaultPadding = 16.0;
       const Duration animationDuration = Duration(milliseconds: 300);
       
       // 枚举
       enum UserStatus { active, inactive, pending }
       
       // 类常量
       class AppConstants {
         static const String appName = 'My App';
         static const int timeout = 30;
       }
    
    4. 文件命名
       // 文件名使用小写蛇形
       user_profile.dart
       custom_button.dart
       api_service.dart
       user_model.dart
       
       // 对应类名
       // user_profile.dart -> class UserProfile
       // custom_button.dart -> class CustomButton
    ''');
  }
}
```

### 3. 注释规范

```dart
// 注释规范
class CommentConventions {
  /*
  注释规范：
  
  1. 文档注释
     - 使用 ///
     - 描述功能
     - 参数说明
     - 返回值说明
  
  2. 行内注释
     - 解释复杂逻辑
     - 避免过度注释
     - 保持更新
  
  3. TODO 注释
     - 标记待办事项
     - 格式：// TODO: 内容
     - 定期清理
  */
  
  void explain() {
    print('''
    注释规范：
    
    1. 文档注释
       /// 获取用户信息
       /// 
       /// 通过用户 ID 获取用户详细信息
       /// 
       /// 参数：
       /// - [id] 用户 ID
       /// 
       /// 返回：
       /// - [Future<User>] 用户对象
       /// 
       /// 异常：
       /// - [ApiException] 网络错误时抛出
       Future<User> getUserById(int id) async {
         // 实现
       }
    
    2. 类文档注释
       /// 用户服务类
       /// 
       /// 提供用户相关的 API 调用方法
       /// 
       /// 示例：
       /// ```dart
       /// final service = UserService();
       /// final user = await service.getUserById(1);
       /// ```
       class UserService {
         // 实现
       }
    
    3. 行内注释
       // 好的注释：解释为什么
       // 使用指数退避策略避免服务器过载
       await Future.delayed(Duration(seconds: retryCount * 2));
       
       // 不好的注释：重复代码
       int count = 0; // 设置 count 为 0
    
    4. TODO 注释
       // TODO: 实现用户认证功能
       // TODO(username): 优化此方法的性能
       // FIXME: 修复这个 bug
       // HACK: 临时解决方案
    
    5. 块注释
       /*
        * 复杂算法说明：
        * 1. 第一步做什么
        * 2. 第二步做什么
        * 3. 第三步做什么
        */
    ''');
  }
}

### 1. 异步编程

```dart
// 异步编程最佳实践
class AsyncBestPractices {
  /*
  异步编程最佳实践：
  
  1. 使用 async/await
     - 简化异步代码
     - 错误处理清晰
     - 易于理解和维护
  
  2. 错误处理
     - 使用 try-catch
     - 提供有意义的错误信息
     - 处理所有错误情况
  
  3. 避免回调地狱
     - 使用 async/await
     - 拆分复杂逻辑
     - 使用 Future.wait
  */
  
  void explain() {
    print('''
    异步编程最佳实践：
    
    1. 使用 async/await
       // 不推荐：回调方式
       void fetchData() {
         api.getData().then((data) {
           setState(() {
             _data = data;
           });
         }).catchError((error) {
           print(error);
         });
       }
       
       // 推荐：async/await
       Future<void> fetchData() async {
         try {
           final data = await api.getData();
           setState(() {
             _data = data;
           });
         } catch (e) {
           print(e);
         }
       }
    
    2. 错误处理
       Future<User> getUser(int id) async {
         try {
           final response = await http.get(Uri.parse('$baseUrl/users/$id'));
           
           if (response.statusCode == 200) {
             return User.fromJson(jsonDecode(response.body));
           } else if (response.statusCode == 404) {
             throw UserNotFoundException('User not found');
           } else {
             throw ApiException('Failed to load user: ${response.statusCode}');
           }
         } on SocketException {
           throw NetworkException('No internet connection');
         } on FormatException {
           throw DataFormatException('Invalid response format');
         }
       }
    
    3. 并行执行
       // 不推荐：顺序执行
       Future<void> loadAllData() async {
         final users = await fetchUsers();
         final products = await fetchProducts();
         final orders = await fetchOrders();
       }
       
       // 推荐：并行执行
       Future<void> loadAllData() async {
         final results = await Future.wait([
           fetchUsers(),
           fetchProducts(),
           fetchOrders(),
         ]);
         
         final users = results[0];
         final products = results[1];
         final orders = results[2];
       }
    
    4. 超时处理
       Future<String> fetchWithTimeout() async {
         try {
           return await http
               .get(Uri.parse(url))
               .timeout(Duration(seconds: 10));
         } on TimeoutException {
           throw TimeoutException('Request timeout');
         }
       }
    ''');
  }
}
```

### 2. 状态管理

```dart
// 状态管理最佳实践
class StateManagementBestPractices {
  /*
  状态管理最佳实践：
  
  1. 最小化状态
     - 只存储必要的状态
     - 避免重复状态
     - 派生状态计算
  
  2. 状态不可变
     - 使用 final
     - 使用 copyWith
     - 避免直接修改
  
  3. 状态提升
     - 提升到合适的层级
     - 通过构造函数传递
     - 通过回调更新
  */
  
  void explain() {
    print('''
    状态管理最佳实践：
    
    1. 最小化状态
       // 不推荐：存储派生状态
       class UserState {
         final String firstName;
         final String lastName;
         final String fullName; // 派生状态，不应存储
       }
       
       // 推荐：计算派生状态
       class UserState {
         final String firstName;
         final String lastName;
         
         String get fullName => '$firstName $lastName';
       }
    
    2. 状态不可变
       // 不推荐：直接修改
       void updateName(String name) {
         _user.name = name; // 直接修改
       }
       
       // 推荐：创建新对象
       void updateName(String name) {
         _user = _user.copyWith(name: name);
       }
       
       // 使用 freezed
       @freezed
       class User with _$User {
         factory User({
           required String name,
           required int age,
         }) = _User;
       }
    
    3. 状态提升
       // 不推荐：状态在子组件
       class Counter extends StatefulWidget {
         @override
         _CounterState createState() => _CounterState();
       }
       
       // 推荐：状态提升
       class ParentWidget extends StatefulWidget {
         @override
         _ParentWidgetState createState() => _ParentWidgetState();
       }
       
       class _ParentWidgetState extends State<ParentWidget> {
         int _count = 0;
         
         @override
         Widget build(BuildContext context) {
           return CounterDisplay(
             count: _count,
             onIncrement: () => setState(() => _count++),
           );
         }
       }
    ''');
  }
}
```

### 3. Widget 最佳实践

```dart
// Widget 最佳实践
class WidgetBestPractices {
  /*
  Widget 最佳实践：
  
  1. 使用 const
     - 编译时常量
     - 避免重建
     - 提高性能
  
  2. 提取 Widget
     - 减少 build 方法复杂度
     - 提高可读性
     - 便于复用
  
  3. 避免深层嵌套
     - 使用方法提取
     - 使用单独的 Widget
     - 保持代码简洁
  */
  
  void explain() {
    print('''
    Widget 最佳实践：
    
    1. 使用 const
       // 不推荐
       @override
       Widget build(BuildContext context) {
         return Column(
           children: [
             Text('Title'),
             Icon(Icons.star),
           ],
         );
       }
       
       // 推荐
       @override
       Widget build(BuildContext context) {
         return Column(
           children: const [
             Text('Title'),
             Icon(Icons.star),
           ],
         );
       }
    
    2. 提取 Widget
       // 不推荐：复杂的 build 方法
       @override
       Widget build(BuildContext context) {
         return Container(
           child: Column(
             children: [
               Row(
                 children: [
                   CircleAvatar(),
                   Column(
                     children: [
                       Text('Name'),
                       Text('Email'),
                     ],
                   ),
                 ],
               ),
               // ... 更多嵌套
             ],
           ),
         );
       }
       
       // 推荐：提取 Widget
       @override
       Widget build(BuildContext context) {
         return Container(
           child: Column(
             children: [
               _buildUserHeader(),
               _buildUserDetails(),
             ],
           ),
         );
       }
       
       Widget _buildUserHeader() {
         return Row(
           children: [
             CircleAvatar(),
             _buildUserInfo(),
           ],
         );
       }
    
    3. 使用命名构造函数
       class CustomButton extends StatelessWidget {
         final String text;
         final VoidCallback onPressed;
         final ButtonStyle style;
         
         const CustomButton({
           Key? key,
           required this.text,
           required this.onPressed,
           required this.style,
         }) : super(key: key);
         
         CustomButton.primary({
           Key? key,
           required String text,
           required VoidCallback onPressed,
         }) : this(
           key: key,
           text: text,
           onPressed: onPressed,
           style: ButtonStyle.primary,
         );
         
         CustomButton.secondary({
           Key? key,
           required String text,
           required VoidCallback onPressed,
         }) : this(
           key: key,
           text: text,
           onPressed: onPressed,
           style: ButtonStyle.secondary,
         );
       }
    ''');
  }
}

## 📖 总结

### 代码规范核心

| 规范 | 描述 | 重要性 |
|------|------|--------|
| **命名规范** | 清晰、一致的命名 | ⭐⭐⭐⭐⭐ |
| **代码结构** | 合理的组织结构 | ⭐⭐⭐⭐ |

| **最佳实践** | 经过验证的做法 | ⭐⭐⭐⭐⭐ |

### 最佳实践总结

1. **命名**：清晰、一致、有意义
2. **结构**：合理组织、易于导航
3. **注释**：解释为什么，而不是做什么
4. **异步**：使用 async/await，完整错误处理
5. **状态**：最小化、不可变、正确提升
6. **Widget**：使用 const、提取、优化

2. **最佳实践**：异步、状态、Widget
