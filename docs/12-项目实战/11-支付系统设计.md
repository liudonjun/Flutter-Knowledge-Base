# 支付系统设计

> Flutter 应用中支付模块的架构、多渠道接入、订单状态与安全对账。

## 模块划分

| 子模块 | 职责 |
| --- | --- |
| **PaymentGateway** | 抽象 `pay(orderId)`，屏蔽渠道 SDK 差异 |
| **订单服务** | 创建订单、查询状态、幂等键 |
| **回调/Webhook** | 服务端接收支付平台异步通知（权威） |
| **客户端结果页** | 展示「处理中/成功/失败」，**不**以 SDK 回调为最终依据 |
| **退款/关单** | 超时未支付关单、售后退款（服务端） |

## 架构

```
features/checkout/
├── presentation/     # 选渠道、确认支付、结果页
├── domain/           # PaymentGateway 接口、PayResult
└── data/             # AlipayGateway、WechatPayGateway、StripeGateway

core/payment/
└── payment_router.dart   # 按 region / 用户选择路由渠道
```

依赖方向：UI → UseCase → Gateway 接口；各 Gateway 实现放 data 层，见 [[46-provider_di详解]]。

## PaymentGateway 抽象

```dart
enum PayStatus { success, cancelled, failed, pending }

class PayResult {
  PayResult({required this.status, this.message});
  final PayStatus status;
  final String? message;
}

abstract class PaymentGateway {
  String get channelId; // alipay | wechat | stripe
  Future<PayResult> pay(String orderId);
}
```

注册示例：

```dart
getIt.registerLazySingleton<PaymentGateway>(
  () => AlipayGateway(getIt()),
  instanceName: 'alipay',
);
```

## 端到端流程

```
1. POST /orders → order_id（业务订单，待支付）
2. 用户选渠道
3. POST /payments/{channel} → 渠道参数（orderInfo / prepay / client_secret）
4. 客户端调 SDK
5. SDK 返回（仅供参考）→ UI 显示处理中
6. 轮询 GET /orders/{id} 或 服务端 webhook 更新 → 成功页
```

**权威状态在后端**；微信 errCode、支付宝 sync 结果均可能不准。

## 订单状态机

```
created → paying → paid
              ↘ cancelled / expired / failed
paid → refunding → refunded（部分/全额）
```

- 创建订单时生成 **幂等 key**（客户端 UUID + 服务端校验）
- `paying` 超时（如 30min）自动 `expired`
- 重复 webhook：按 `transaction_id` 去重

## 安全与合规

1. **私钥/Secret 仅服务端**；客户端无签名逻辑
2. **金额以后端为准**；客户端只展示，提交只传 `order_id`
3. **HTTPS + 证书校验**；支付相关接口鉴权
4. **PCI**：卡号走 Stripe SDK，勿自建表单存卡
5. **日志**：生产禁用 [[43-pretty_dio_logger详解]] 打印支付 body

## 异常处理

| 场景 | 客户端 | 服务端 |
| --- | --- | --- |
| 用户取消 | `PayStatus.cancelled` | 订单仍待支付 |
| 已支付重复点 | 查单已是 paid → 成功页 | 幂等返回 |
| 网络中断 | 处理中 + 轮询 | webhook 最终一致 |
| 金额不一致 | 不展示成功 | 拒绝 notify |
