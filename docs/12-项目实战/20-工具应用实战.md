# 工具应用实战

> 计算器、扫描、效率类等「工具型」Flutter 应用的设计要点。

## 产品特点

- 功能 **单一或少量**，交互路径短。
- 重视启动速度、离线可用、低权限。
- 常需 Widget、快捷方式（平台相关）。

---

## 架构建议

小项目可用 **简化分层**：

```
lib/
├── main.dart
├── app/
├── features/        # 按工具功能划分
├── services/        # 存储、权限、分享
└── shared/widgets/
```

状态：`Riverpod` / 简单 `ChangeNotifier` 即可，避免过度工程。
