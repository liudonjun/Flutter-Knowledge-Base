# 消息系统设计

> 站内信、推送与实时消息的整体架构：在线推送、离线到达、存储与 Deep Link。

## 模块划分

| 子模块 | 职责 |
| --- | --- |
| **实时通道** | WebSocket / Socket.IO — [[41-socket_io_client详解]]、[[26-web_socket_channel详解]] |
| **推送通道** | FCM/APNs — [[24-firebase_messaging详解]] |
| **本地展示** | 前台横幅 — [[25-flutter_local_notifications详解]] |
| **存储** | 会话、消息、未读 — SQLite / Hive |
| **同步** | 增量拉取、已读上报、多端一致 |

## 架构总览

```
                    ┌─────────────┐
  在线 ────────────►│ WS / Socket │──► 写本地 DB ──► UI
                    └─────────────┘
                           ▲
  离线 ── FCM ─────────────┘（唤醒 / 拉增量）
         │
         └──► flutter_local_notifications（前台可选）
```

**原则**：消息 **持久化以服务端为准**；客户端 DB 是缓存；UI 读本地 + 订阅流。

## 数据模型

```dart
class Conversation {
  Conversation({
    required this.id,
    required this.title,
    required this.updatedAt,
    this.unread = 0,
    this.lastPreview = '',
  });

  final String id;
  final String title;
  final DateTime updatedAt;
  final int unread;
  final String lastPreview;
}

class ChatMessage {
  ChatMessage({
    required this.id,
    required this.conversationId,
    required this.senderId,
    required this.content,
    required this.createdAt,
    this.status = MessageStatus.sent,
  });

  final String id;
  final String conversationId;
  final String senderId;
  final String content;
  final DateTime createdAt;
  final MessageStatus status;
}

enum MessageStatus { pending, sent, delivered, read, failed }
```

- 客户端生成 `client_msg_id`（UUID），与服务端 `id` 映射，防重连重复
- 未读：会话级计数；进入聊天页批量 `mark_read`

## 实时层

```dart

```

实现可切换 [[41-socket_io_client详解]] 或 [[26-web_socket_channel详解]]；接口保持一致便于测试。

### 重连策略

1. 断线 → 指数退避重连（上限 30s）
2. 重连成功 → `GET /messages?since=last_seq` 拉缺口
3. Token 401 → 刷新后再连

## 推送层

| 状态 | 行为 |
| --- | --- |
| 前台 + 当前会话 | 仅更新 UI，可不弹通知 |
| 前台 + 其他会话 | [[25-flutter_local_notifications详解]] |
| 后台 | FCM 系统通知 |
| 点击通知 | Deep Link → `/chat/{conversation_id}` |

FCM `data`  payload 示例：

```json
{ "conversation_id": "c_123", "type": "chat", "msg_id": "m_456" }
```

绑定 Token：`POST /users/me/fcm-token` — 见 [[24-firebase_messaging详解]]。

## 本地存储

- **会话列表**：按 `updatedAt` 排序索引
- **消息表**：`(conversation_id, created_at)` 复合索引
- **分页**：聊天页 `reverse` 列表向上加载 `before_id`

登出清库或只清用户相关表；换账号必须隔离 DB 文件名或前缀。

## 未读与同步

```
收到消息 → unread++（非当前会话且非本人）
进入会话 → unread=0 + POST /conversations/{id}/read
多端已读 → WS 事件 read_receipt → 更新本地 status
```

服务端维护 authoritative unread；客户端乐观更新，失败回拉会话列表。

## 安全

1. WS 连接必须鉴权，Topic/房间校验 membership
2. 推送 payload **不含** 消息明文（敏感应用用「您有一条新消息」）
3. 附件 URL 用短期 signed URL

## 测试建议

| 用例 | 说明 |
| --- | --- |
| 发送 pending → sent | 断网发消息，恢复后 ack |
| 重连补消息 | 模拟 since 增量 |
| 推送点击路由 | 冷启动 / 热启动进正确会话 |
| 未读一致性 | 多端已读同步 |
