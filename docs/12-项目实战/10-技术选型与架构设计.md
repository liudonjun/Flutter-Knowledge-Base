```dart
class LayeredArchitecture {
  /*
  分层架构：
  
  1. 经典三层
     - 表示层 (Presentation)
     - 业务层 (Business)
     - 数据层 (Data)
  
  2. Clean Architecture
     - Entities
     - Use Cases
     - Interface Adapters
     - Frameworks & Drivers
  
  3. 实践建议
     - 单向依赖
     - 关注点分离
     - 可测试性
  */
  
  void explain() {
    print('''
    分层架构：
    
    1. 经典三层架构
       lib/
         presentation/      # 表示层
           screens/
           widgets/
           viewmodels/
         business/          # 业务层
           models/
           services/
           repositories/
         data/              # 数据层
           datasources/
           repositories/
           models/
    
    2. Clean Architecture
       lib/
         core/              # 核心层
           errors/
           usecases/
         features/          # 功能层
           feature1/
             data/
             domain/
             presentation/
           feature2/
             data/
             domain/
             presentation/
    
    3. 依赖规则
       // 外层依赖内层
       Presentation → Domain ← Data
       
       // 接口隔离
       abstract class UserRepository {
         Future<User> getUser(int id);
       }
       
       // 实现细节在 Data 层
       class UserRepositoryImpl implements UserRepository {
         final RemoteDataSource remote;
         final LocalDataSource local;
         
         @override
         Future<User> getUser(int id) async {
           try {
             return await remote.getUser(id);
           } catch (e) {
             return await local.getUser(id);
           }
         }
       }
    ''');
  }
}
```

### 2. 模块化设计

```dart
// 模块化设计
class ModularDesign {
  /*
  模块化设计：
  
  1. 模块划分
     - 按功能划分
     - 按业务划分
     - 按层级划分
  
  2. 模块通信
     - 接口定义
     - 依赖注入
     - 事件总线
  
  3. 优势
     - 代码复用
     - 独立开发
     - 易于测试
  */
  
  void explain() {
    print('''
    模块化设计：
    
    1. 模块划分
       lib/
         core/              # 核心模块
           di/
           network/
           storage/
         features/          # 功能模块
           auth/
             auth_module.dart
             data/
             domain/
             presentation/
           home/
             home_module.dart
             data/
             domain/
             presentation/
           profile/
             profile_module.dart
             data/
             domain/
             presentation/
         shared/            # 共享模块
           widgets/
           utils/
           constants/
    
    2. 模块接口
       // 定义模块接口
       abstract class AuthModule {
         Future<User> login(String email, String password);
         Future<void> logout();
         bool get isLoggedIn;
       }
       
       // 模块实现
       class AuthModuleImpl implements AuthModule {
         final AuthService _service;
         
         AuthModuleImpl(this._service);
         
         @override
         Future<User> login(String email, String password) {
           return _service.login(email, password);
         }
       }
    
    3. 模块注册
       class AppModule {
         static void setup() {
           // 注册模块
           getIt.registerSingleton<AuthModule>(
             AuthModuleImpl(getIt<AuthService>()),
           );
         }
       }
    ''');
  }
}
```

## 📖 总结

### 技术选型决策树

```
1. 状态管理选择
   - 简单应用 → Provider/GetX
   - 复杂应用 → Bloc/Riverpod
   - 快速开发 → GetX

2. 网络库选择
   - 简单需求 → http
   - 复杂需求 → Dio
   - 类型安全 → Chopper

3. 数据库选择
   - 配置存储 → SharedPreferences
   - 结构化数据 → SQLite/Isar
   - 高性能 → Isar
```

### 架构设计原则

1. **单一职责**：每个模块只负责一个功能
2. **依赖倒置**：依赖抽象而非具体实现
3. **接口隔离**：使用小而专一的接口
4. **开闭原则**：对扩展开放，对修改关闭
