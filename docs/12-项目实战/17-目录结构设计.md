# 目录结构设计

> Flutter 项目 `lib/` 目录组织与 feature-first 实践。

## 推荐结构（中型项目）

```
lib/
├── main.dart
├── app/                    # MaterialApp、路由、主题
├── core/                   # 常量、错误、网络客户端、DI
├── features/
│   ├── auth/
│   │   ├── data/
│   │   ├── domain/
│   │   └── presentation/
│   └── home/
│       └── ...
└── shared/                 # 跨 feature 的 widget、utils
```

**依赖方向**：`presentation → domain ← data`，`features` 之间不互相 import。

---

## 与分层架构对应

| 目录 | 职责 |
| --- | --- |
| `data` | DTO、API、本地数据源、Repository 实现 |
| `domain` | Entity、Repository 接口、UseCase |
| `presentation` | Page、Widget、Controller/Bloc |

详见 [[01-项目架构设计]]、[[20-架构设计模式详解]]。

---

## 命名与文件

- 页面：`login_page.dart` 或 `login_screen.dart`（团队统一即可）
- 一个文件一个 public Widget（大文件按 private 组件拆分）
- 测试镜像：`test/features/auth/...`

---

## 反模式

- 按类型分 `widgets/`、`pages/`、`models/` 全局大杂烩（小项目可接受，大了难维护）
- `utils.dart` 万能垃圾场——按领域拆 `date_formatter.dart` 等
