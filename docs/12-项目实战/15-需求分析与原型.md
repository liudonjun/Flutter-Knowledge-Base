# 需求分析与原型

> Flutter 项目启动：从需求澄清、用户故事到可评审原型与技术预研。

## 流程概览

```
业务目标 → 用户故事 → 功能边界 → 原型/线框 → 技术预研 → 迭代 backlog
```

与 [[10-技术选型与架构设计]]、[[01-项目架构设计]] 衔接：原型确认后再定架构与分包。

## 用户故事（模板）

```text
作为 [角色]，我希望 [能力]，以便 [价值]。

验收标准：
- Given ... When ... Then ...
- 异常：网络失败时 ...
```

示例：

```text
作为 买家，我希望 在购物车修改数量，以便 下单前调整商品数。
- 数量为 0 时移除商品
- 超库存时提示并回滚
```

## 功能边界（In / Out）

| In Scope | Out of Scope（V1 不做） |
| --- | --- |
| 邮箱登录 | 生物识别 |
| 支付宝 + 微信 | 国际 PayPal |
| 商品列表分页 | 推荐算法 |

明确 **Out** 可避免原型无限膨胀。

## 原型工具

| 工具 | 适用 |
| --- | --- |
| Figma / Sketch | 高保真、设计系统 |
| 纸笔 / Excalidraw | 快速评审流程 |
| Flutter 原型 | 验证动画、手势、性能 |

Flutter 原型可复用 [[53-Material组件详解]]，验证后再拆 `features/` — [[17-目录结构设计]]。

## 页面清单与导航

产出 **站点地图**：

```text
Splash → Login → Main(Tab: Home | Cart | Profile)
Home → ProductDetail → Checkout → PayResult
```

对应路由表预研：go_router 路径与 deep link — [[01-导航与路由基础]]。

## 技术预研（Spike）

在承诺排期前，对高风险项做 1–3 天 Spike：

| 风险 | 预研 |
| --- | --- |
| 微信/支付宝 | [[33-alipay详解]]、[[34-wechat_pay详解]] 沙箱 |
| 实时 IM | [[41-socket_io_client详解]] 连测试服 |
| 地图 | [[58-geolocator详解]]、[[18-地图系统集成]] |
| 推送 | [[24-firebase_messaging详解]] 真机 |

结论写入 ADR 或 [[10-技术选型与架构设计]] 附录。

## 非功能需求

- **性能**：首屏时间、列表帧率目标
- **兼容**：最低 iOS / Android 版本
- **安全**：登录、支付、隐私合规
- **离线**：哪些页面可离线浏览
