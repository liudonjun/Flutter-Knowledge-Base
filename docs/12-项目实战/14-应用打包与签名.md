# 应用打包与签名

> Flutter Android / iOS 构建产物、签名配置与上架前检查清单。

## 产物类型

| 平台 | 产物 | 用途 |
| --- | --- | --- |
| **Android** | APK | 侧载、部分渠道 |
| **Android** | **AAB** | Google Play 必须 |
| **iOS** | IPA | App Store / TestFlight |
| **Web** | `build/web` | 静态托管 |

Flutter 命令：

```bash
flutter build apk --release
flutter build appbundle --release
flutter build ipa --release
```

## 版本号

`pubspec.yaml`：

```yaml
version: 1.2.3+45   # 显示版本 + build number
```

| 平台 | 映射 |
| --- | --- |
| Android | `versionName` / `versionCode`（+45） |
| iOS | `CFBundleShortVersionString` / `CFBundleVersion` |

每次上架 **build number 必须递增**。

## Android 签名

### 生成 keystore

```bash
keytool -genkey -v -keystore upload-keystore.jks -keyalg RSA -keysize 2048 -validity 10000 -alias upload
```

### key.properties（勿提交 Git）

```properties
storePassword=***
keyPassword=***
keyAlias=upload
storeFile=../upload-keystore.jks
```

`android/app/build.gradle` 引用 signingConfigs release。

### 检查项

- `applicationId` 与支付/微信开放平台一致
- `minSdk` / `targetSdk` 满足商店与插件要求
- ProGuard/R8：`flutter_stripe` 等按文档 keep 规则
- 64K 方法数：启用 multidex（通常 Flutter 模板已处理）

## iOS 签名

1. Apple Developer 创建 App ID、Provisioning Profile
2. Xcode → Signing & Capabilities：Team、Bundle ID
3. 能力开关：Push、Sign in with Apple、Associated Domains 等与功能一致
4. Archive：Product → Archive → Distribute

### 常见能力

| 功能 | Capability |
| --- | --- |
| 推送 | Push Notifications |
| 微信/支付宝回跳 | URL Types + Universal Links |
| Apple 登录 | Sign in with Apple |

## 构建优化（Release）

```bash
flutter build appbundle --release --obfuscate --split-debug-info=build/symbols
```

- 混淆减小体积；符号文件单独存档用于 crash 还原
- 移除 debug 日志与 [[43-pretty_dio_logger详解]]

## 上架前清单

- [ ] Release 包真机全流程（登录、支付、推送）
- [ ] 隐私政策 URL、权限说明文案
- [ ] 应用图标、截图、商店描述
- [ ] 第三方 SDK 隐私清单（Android Data safety / iOS Privacy Manifest）
- [ ] 版本号、渠道号正确
- [ ] 无 test api key / 调试开关

## 渠道包（Android）

多渠道可在 `productFlavors` 注入 `channel` meta-data，或使用 Walle 等方案；**签名共用同一 upload key**。
