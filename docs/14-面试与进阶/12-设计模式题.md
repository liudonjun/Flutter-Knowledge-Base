# 设计模式题

> Flutter 项目中常见设计模式的应用场景与面试答法。

## 单例与工厂

**单例**：`GetIt` 注册全局服务、数据库连接（注意测试可 reset）。

```dart
class ApiClient {
  ApiClient._();
  static final instance = ApiClient._();
}
```

**工厂**：根据类型创建不同 Repository 实现；JSON 反序列化 `fromJson` 也是工厂思想。

面试：单例缺点——隐藏依赖、难测试；应用 DI 容器替代滥用单例。

## 观察者 / 发布订阅

Flutter 中无处不在：

- `ChangeNotifier` + `ListenableBuilder`
- `Stream` / BLoC 的 Event → State
- `WidgetsBindingObserver` 监听生命周期

**答法**：Observer 解耦发布者与订阅者；注意 dispose 取消订阅防泄漏。

## 策略与装饰

**策略**：支付方式（微信/支付宝/Stripe）实现同一 `PaymentStrategy` 接口。

**装饰**：Dio 拦截器链——日志、Token、重试层层包装。

```dart
abstract class PaymentStrategy {
  Future<void> pay(Order order);
}
```

## Repository 模式

**职责**：对 UI/Bloc 隐藏数据来源（网络 / 本地 / 缓存）。

```dart
abstract class UserRepository {
  Future<User> getUser(String id);
}

class UserRepositoryImpl implements UserRepository {
  UserRepositoryImpl(this._api, this._cache);
  final ApiClient _api;
  final LocalCache _cache;

  @override
  Future<User> getUser(String id) async {
    final cached = await _cache.getUser(id);
    if (cached != null) return cached;
    final user = await _api.fetchUser(id);
    await _cache.saveUser(user);
    return user;
  }
}
```

面试：Repository 便于单测 Mock、统一缓存与错误处理。详见 [[20-架构设计模式详解]]。

## 回答技巧

- 每个模式：**场景 → 代码结构 → 优缺点**。
- 结合 Flutter：InheritedWidget 是装饰/依赖注入的变体；BLoC 是 Observer 的架构化。
