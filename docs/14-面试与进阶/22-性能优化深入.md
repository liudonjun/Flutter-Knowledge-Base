# 性能优化深入

> Timeline 分析、Impeller/Skia、内存泄漏与包体积的系统化方法。

## Timeline 分析

1. 打开 DevTools → **Performance**，录一段交互。
2. 看 **UI / Raster** 线程是否超过帧预算（16.6ms @60Hz）。
3. 展开 **Frame**，定位 `BUILD`、`LAYOUT`、`PAINT` 热点 Widget。
4. 开启 **Track Widget rebuilds**（debug 标志）配合 [[06-性能优化面试题]]。

---

## Impeller / Skia

- **Shader jank**：Skia 运行时编译导致首动画卡顿；Impeller 预编译缓解。
- **RepaintBoundary**：隔离动画区域，减少无效重绘。
- **Layer 过多**：过多 Opacity/Clip 会增加合成成本。

详见 [[27-编译原理与引擎]]。

---

## 内存泄漏排查

常见泄漏源：

| 源 | 处理 |
| --- | --- |
| `AnimationController` | `dispose()` |
| `StreamSubscription` | `cancel()` |
| `TextEditingController` | `dispose()` |
| 静态变量持有 `BuildContext` | 避免 |

DevTools **Memory** → Snapshot diff，查 retained 对象链。
