# 混合与原生开发

> Add-to-App、Platform Channel 与原生团队协作要点。

## 混合栈场景

| 模式 | 说明 |
| --- | --- |
| **纯 Flutter** | 整 app Flutter，最常见 |
| **Add-to-App** | 原生 app 嵌入 Flutter 模块（单 Activity/ViewController） |
| **Flutter + WebView** | H5 活动页，注意性能与安全 |
| **多引擎** | 多 FlutterEngine 实例，内存开销大，慎用 |

**[[混合开发]]** 选型：存量原生大、Flutter 只做新模块时用 Add-to-App。

---

## Platform Channel

```dart
static const _channel = MethodChannel('com.example/battery');

Future<int> getBatteryLevel() async {
  final level = await _channel.invokeMethod<int>('getBatteryLevel');
  return level ?? -1;
}
```

- **MethodChannel**：一次性调用
- **EventChannel**：原生向 Dart 推流
- **FFI**：高性能 C/C++ 库

注意：Channel 在主线程，耗时操作放原生后台线程。

## 原生开发互补

**[[原生开发]]** 仍需掌握：

- 生命周期、权限、推送、应用商店配置
- iOS：Info.plist、Signing；Android：Gradle、ProGuard

Flutter 开发者不必写大量原生 UI，但要能 **读原生日志、对接 SDK**。
