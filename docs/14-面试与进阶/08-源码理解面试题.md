# 源码理解面试题

> Framework / Engine 渲染管线、Binding、Element 与 Platform Channel 的高频追问。

## 三棵树

**题：Widget、Element、RenderObject 关系？**

| 层 | 职责 |
| --- | --- |
| **Widget** | 配置 immutable，轻量 |
| **Element** | 挂载点，持有 State，负责 update |
| **RenderObject** | layout / paint，heavy |

**答**：`setState` → mark element dirty → `build` 新 Widget → `updateChild` 比较 runtimeType/key → **reuse** 或 **inflate** 新 Element。

详见 [[01-Flutter渲染原理]]、[[00-一切皆Widget详解]]。

---

## Binding 初始化

**题：`WidgetsFlutterBinding.ensureInitialized()` 做什么？**

**答**：确保 Flutter 绑定初始化（Scheduler、Services、Gesture 等），`runApp` 前必须；插件在 async main 里调 native 前也要调用。

**题：启动流程概略？**

`main` → binding → `runApp` → `attachRootWidget` → build/layout/paint → 首帧。

---

## Element 更新机制

**题：`setState` 为什么不能在 build 里调？**

重建过程中 mark dirty 会导致断言/无限循环。

**题：GlobalKey 如何工作？**

GlobalKey 在全局注册 Element，可 `currentState`/`currentContext`；跨树移动时 **reparent** Element 保留 State。

---

## Layer 与合成

**题：RepaintBoundary 原理？**

创建独立 **Layer**，子树重绘不必然上传父层；过度使用增加内存与 layer 数。

**题：UI 线程和 Raster 线程？**

Dart build/layout/paint 在 UI isolate；光栅化可在 Raster 线程；Jank 可能来自任一侧 — [[06-性能优化面试题]]。

---

## Platform Channel

**题：MethodChannel 流程？**

```
Dart invokeMethod → codec 编码 → engine → 平台 handler → 结果回调
```

**答**：异步；主线程限制因平台而异；高频大数据用 EventChannel / FFI。

**题：Plugin 注册？**

GeneratedPluginRegistrant 在引擎启动注册各插件。

见 [[01-平台集成指南]]。

---

## 框架源码阅读方法

1. **从 API 点进去**：如 `ListView.builder` → `SliverList`
2. **调试**：Flutter DevTools + 断点 `Element.rebuild`
3. **画管线**：constraints → layout → paint → compositing
4. **不必背源码**：说清 **设计意图** 与 **扩展点**（如 `RenderObjectWidget`）

---

## 进阶追问

**题：InheritedElement 如何通知依赖？**

注册依赖的 Element 列表；notify 时 markNeedsBuild。

**题：Hot Reload 限制？**

改 class 字段、enum、generic 等需 Hot Restart。

**题：Impeller / Skia？**

Skia 传统 2D 引擎；Impeller 预编译 shader 减首帧 jank — [[04-Flutter引擎架构]]。
