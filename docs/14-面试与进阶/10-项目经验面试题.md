# 项目经验面试题

> 如何用项目经历回答复杂度、协作、冲突与结果类问题。

## 项目背景与目标

**常见问题**：「介绍一个最有挑战的项目。」

答题框架（配合 [[09-STAR法则详解]]）：

1. 业务与用户价值（1 句）
2. 技术栈与规模（Flutter 版本、模块数、团队人数）
3. 你负责的范围

避免：从公司创立讲起；堆砌与岗位无关的技术名词。

---

## 个人职责边界

**追问**：「这是你一个人做的吗？」

诚实说明：

- 独立负责：例如「列表模块从 0 到 1」
- 协作：与 iOS/Android/后端/设计的接口与联调节点
- 领导/评审：Code Review、方案评审中的角色

---

## 技术难点与取舍

高频难点类型：

| 类型 | 示例答法 |
| --- | --- |
| **性能** | 列表 jank、内存泄漏、启动慢——工具 + 指标 |
| **状态** | 多 Tab 状态保持、跨页共享——选型理由 |
| **原生** | Platform Channel、推送、支付 SDK 集成 |
| **交付** |  deadline 紧——砍 scope、分阶段、Feature Flag |

强调：**为什么选 A 不选 B**，展示工程判断。

---

## 量化结果

尽量准备：

- 性能：FPS、启动时间、包体积、崩溃率
- 业务：转化率、留存、开发效率（发版周期）
- 质量：测试覆盖率、线上 bug 数

没有数据：说明如何建立监控、A/B 或下一版验证计划。

---

## 常见陷阱题

- **失败经历**：真实失败 + 复盘 + 改进（不要假失败装聪明）。
- **与同事分歧**：对事不对人，数据/实验说话。
- **为何离职 / 为何 Flutter**：积极、与职业规划一致。
