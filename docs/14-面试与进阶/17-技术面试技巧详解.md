# 技术面试技巧

> 笔试、现场编码与远程面试的通用准备方法与节奏控制。

## 面试前调研

1. **公司与产品**：业务域、Flutter 在栈中的位置（纯 Flutter / 混合 / 维护型）。
2. **JD 关键词**：状态管理、性能、原生桥接——准备对应故事。
3. **题库对齐**：复习 [[02-Flutter基础面试题]]、[[16-状态管理面试题]]、[[06-性能优化面试题]]。
4. **材料准备**：简历项目一页纸、可演示 APK/TestFlight、架构图。

---

## 白板 / 共享编码

| 阶段 | 做法 |
| --- | --- |
| **读题** | 复述需求、确认边界与输入输出 |
| **方案** | 先讲思路再写代码，说明复杂度 |
| **实现** | 优先可运行最小版本，再优化 |
| **自测** | 口述 2～3 个用例，包括边界 |

Flutter 现场题常见：计数器、列表分页、简单状态管理、布局题（Flex 约束）。

**避免**：长时间沉默；不经思考直接写；忽略 null 与 dispose。

---

## 沟通节奏

- **结构化**：结论 → 理由 → 例子（可配合 [[15-问题回答策略]]）。
- **控制时长**：基础题 2～3 分钟，项目题 5～8 分钟，留追问空间。
- **不会时**：说明已知部分 + 学习路径，不要编造。
- **反问环节**：团队 Flutter 版本、CI、代码规范、性能指标——体现主动性。

---

## 复盘与总结

每次面试后记录：

- 被问到的知识点与薄弱项
- 编码题是否卡在语法还是思路
- 哪些项目故事讲得不清楚
