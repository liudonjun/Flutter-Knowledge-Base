# 性能优化面试题

> 帧率、重建、列表与内存相关的高频面试题与答题要点。

## jank 排查思路

1. **DevTools Performance**：看 UI / Raster 线程是否超 16.6ms（60fps）。
2. **Timeline**：定位 `build`、`layout`、`paint` 哪一阶段耗时。
3. **Repaint Rainbow**：检查是否大面积重绘。
4. **常见原因**：build 里重计算、大图解码、同步 I/O、Shader 编译（首帧）。

**答法**：先度量再优化，给出一次真实优化案例（如列表卡顿 → `ListView.builder` + `itemExtent`）。

---

## 重建与重绘优化

| 手段 | 说明 |
| --- | --- |
| `const` 构造函数 | 子树可跳过 rebuild |
| 拆分 Widget | 把变化部分拆成独立 Stateful/Consumer |
| `RepaintBoundary` | 隔离高频动画区域 |
| `ListenableBuilder` / `Selector` | 只监听必要字段 |
| 避免 build 副作用 | 不在 build 里创建 Controller/Future |

**ListView vs ListView.builder**：前者一次性 build 全部 child；后者懒加载，长列表必用 builder + 稳定 `key`/`itemExtent`。

---

## 图片与列表优化

- **图片**：指定 `cacheWidth`/`cacheHeight`；WebP；占位与错误图；CDN 尺寸适配。
- **列表**：`AutomaticKeepAliveClientMixin` 保留 Tab 滚动位置；分页加载；避免 `shrinkWrap: true` 嵌大列表。
- **反模式**：`IntrinsicHeight` 滥用、无界嵌套 ScrollView。

---

## DevTools 使用

- **CPU Profiler**：找热点 Dart 方法。
- **Memory**：查泄漏（未 dispose 的 Controller、Stream 订阅）。
- **Network**：接口耗时与并发。

---

## 其他高频题

**Flutter 性能优化总纲**：减少 rebuild → 优化 layout/paint → 控制内存 → 启动与包体积。

**如何优化启动**：延迟初始化、减少 main 同步 work、`runApp` 前必要插件并行、Splash 占位。
