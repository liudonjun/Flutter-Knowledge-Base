# 项目展示详解

> 面试中演示 App、架构图与关键链路的表达方法。

## Demo 准备

- **真机优先**：避免模拟器网络/性能意外。
- **黄金路径**：30～60 秒内展示核心价值（登录 → 核心功能 → 亮点）。
- **离线兜底**：预录屏备份，防止现场网络失败。
- **Debug 关**：用 profile/release 包，避免 debug 横幅与卡顿。

---

## 架构一页纸

建议包含：

```
[客户端 Flutter]
   Presentation (Bloc/Riverpod)
        ↓
   Domain (UseCase / Entity)
        ↓
   Data (Repository → API / DB)
        ↕
[后端] REST / GraphQL
```

标注：状态管理选型、缓存策略、关键第三方库（[[08-retrofit详解]] 等）。

可链 [[01-项目架构设计]]、[[20-架构设计模式详解]]。

## 亮点故事线

用 [[09-STAR法则详解]] 准备 2～3 个故事：

1. **性能优化**（指标前后对比）
2. **复杂状态 / 离线**（技术难点）
3. **跨端 / 原生集成**（Channel、推送）

每条 3 分钟口述 + 可指 Demo 对应页面。
