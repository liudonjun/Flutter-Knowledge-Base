# 持续学习指南

> 学习计划、知识管理、效率方法与 Flutter 新特性跟进。

## 学习计划

**SMART 原则**：具体技能 + 时间盒 + 可验证产出。

示例 4 周计划：

| 周 | 目标 | 产出 |
| --- | --- | --- |
| 1 | Riverpod 2.x | 小 demo + 笔记 |
| 2 | 性能 DevTools | 优化一个列表页并截图 Timeline |
| 3 | 源码 Binding | 一篇 [[18-Flutter源码学习]] 读书笔记 |
| 4 | 模拟面试 | 完成 [[23-代码面试详解]] 里 5 道题 |

---

## 知识库 / Obsidian

- 一概念一笔记，双向链接 `[[Widget]]` 与本站一致。
- 模板：问题 → 答案要点 → 代码 → 相关链接。
- 定期回顾：Weekly 扫「待处理」与 [[handbook/wiki-link-backlog]]（若维护站点）。

**[[知识管理]]**：PARA 或按模块（Widget / 状态 / 网络）分文件夹。

---

## 新特性跟进

1. [Flutter release notes](https://docs.flutter.dev/release/release-notes)
2. [Dart changelog](https://github.com/dart-lang/sdk/blob/main/CHANGELOG.md)
3. Flutter Forward / Google I/O 关键 session

**[[新特性学习]]**：Major 版本升级先在 branch 试 `flutter upgrade`，跑 `dart fix --apply`。

---

## 学习社区

- Flutter Discord / 中文社区 / 本地 Meetup
- GitHub Discussions、Stack Overflow
- **[[学习社区]]**：提问前先最小复现；回答时贴版本号 `flutter doctor -v`

**[[学习效率]]**：

- 番茄钟 + 刻意练习（写代码而非只收藏）
- 教别人（费曼技巧）巩固理解

## 技术趋势与行业动态

索引/wiki：`[[技术趋势]]`、`[[行业动态]]`、`[[未来方向]]`。

| 维度 | 关注什么 | 行动 |
| --- | --- | --- |
| **Flutter 生态** | Impeller、Wasm、桌面/Web 成熟度 | 跟 release notes，在 side project 试新 API |
| **跨端格局** | Compose Multiplatform、RN、鸿蒙 | 读 [[21-跨平台对比详解]]，明确团队选型边界 |
| **AI + 开发** | Copilot、代码生成、测试辅助 | 用于脚手架与文档，核心逻辑仍手写 + Review |
| **行业应用** | 金融科技合规、出海、IoT 嵌入式 | 选 1 个垂直领域读案例，面试能讲业务约束 |

**面试/规划话术**：不堆砌 buzzword；用「我如何验证趋势 → 小范围试点 → 度量结果」说明判断力。职业路径见 [[25-职业发展与领导力#技术趋势判断]]。
