# Flutter 项目结构详解

> 深入理解 Flutter 项目结构，建立良好的代码组织习惯。

## 📁 项目结构概述

### 1. 标准项目结构
```
flutter_project/
├── android/              # Android 平台代码
├── ios/                  # iOS 平台代码
├── web/                  # Web 平台代码
├── windows/              # Windows 平台代码
├── macos/                # macOS 平台代码
├── linux/                # Linux 平台代码
├── lib/                  # Dart 源代码
├── test/                 # 测试代码
├── assets/               # 资源文件
├── pubspec.yaml          # 项目配置文件
├── pubspec.lock          # 依赖锁定文件
├── README.md             # 项目说明
├── LICENSE               # 许可证文件
├── .gitignore            # Git 忽略文件
├── .metadata             # Flutter 元数据
├── analysis_options.yaml # 代码分析配置
└── l10n.yaml             # 国际化配置
```

### 2. lib 目录结构详解
```
lib/
├── main.dart             # 应用入口
├── app.dart              # 应用配置
├── config/               # 配置文件
│   ├── api_config.dart
│   ├── theme_config.dart
│   └── routes_config.dart
├── core/                 # 核心功能
│   ├── constants/        # 常量定义
│   ├── errors/           # 错误处理
│   ├── network/          # 网络层
│   ├── storage/          # 存储层
│   └── utils/            # 工具类
├── data/                 # 数据层
│   ├── models/           # 数据模型
│   ├── repositories/     # 数据仓库
│   ├── datasources/      # 数据源
│   └── providers/        # 状态管理
├── domain/               # 领域层
│   ├── entities/         # 实体类
│   ├── repositories/     # 仓库接口
│   └── usecases/         # 用例
├── presentation/         # 表现层
│   ├── pages/            # 页面
│   ├── widgets/          # 自定义 Widget
│   ├── blocs/            # BLoC 状态管理
│   └── providers/        # Provider 状态管理
├── routes/               # 路由配置
│   ├── app_router.dart
│   └── route_names.dart
└── l10n/                 # 国际化文件
    ├── app_en.arb
    ├── app_zh.arb
    └── l10n.yaml
```

```
│   └── usecases/         # 基础用例

│   ├── models/           # 数据模型

│   └── datasources/      # 数据源
├── domain/               # 领域层
│   ├── entities/         # 实体类
│   ├── repositories/     # 仓库接口
```

## 📝 文件命名规范

### 1. 文件命名规则
```dart
// 文件命名: snake_case
// 好的命名
home_screen.dart
user_model.dart
api_service.dart
custom_button.dart
constants.dart

// 不好的命名
HomeScreen.dart
UserModel.dart
apiService.dart
custom-button.dart
Constants.dart
```

### 2. 目录命名规则
```dart
// 目录命名: snake_case
// 好的命名
lib/
├── screens/
├── widgets/
├── models/
├── services/
└── utils/

// 不好的命名
lib/
├── Screens/
├── Widgets/
├── Models/
├── Services/
└── Utils/
```

### 3. 类命名规则
```dart
// 类命名: PascalCase
// 好的命名
class HomeScreen extends StatelessWidget {}
class UserModel {}
class ApiService {}
class CustomButton extends StatelessWidget {}

// 不好的命名
class home_screen extends StatelessWidget {}
class user_model {}
class api_service {}
class custom_button extends StatelessWidget {}
```

## 🎨 资源文件管理

### 1. 资源目录结构
```
assets/
├── images/               # 图片资源
│   ├── icons/            # 图标
│   ├── logos/            # Logo
│   ├── backgrounds/      # 背景图
│   └── photos/           # 照片
├── fonts/                # 字体资源
│   ├── Roboto/
│   └── CustomFont/
├── data/                 # 数据文件
│   ├── json/             # JSON 数据
│   └── mock/             # 模拟数据
└── animations/           # 动画资源
    ├── lottie/           # Lottie 动画
    └── rive/             # Rive 动画
```

### 2. 资源引用方式
```dart
// 图片资源
Image.asset('assets/images/logo.png')
Image.asset('assets/images/icons/star.png', width: 24, height: 24)

// 字体资源
Text(
  '自定义字体',
  style: TextStyle(
    fontFamily: 'CustomFont',
    fontSize: 18,
  ),
)

// JSON 数据
import 'dart:convert';

Future<Map<String, dynamic>> loadJsonData() async {
  final String jsonString = await rootBundle.loadString('assets/data/config.json');
  return jsonDecode(jsonString);
}
```

### 3. 资源优化
```yaml
# pubspec.yaml 资源配置
flutter:
  assets:
    # 单个文件
    - assets/images/logo.png
    
    # 整个目录
    - assets/images/
    
    # 子目录
    - assets/images/icons/
    
    # 带变体的资源
    - path: assets/images/logo.png
      flavors:
        - dev
        - prod
```

## 🎯 最佳实践总结

### 1. 目录组织
```dart
// 推荐的项目结构
lib/
├── main.dart             # 应用入口
├── app.dart              # 应用配置
├── config/               # 配置文件
├── core/                 # 核心功能
├── data/                 # 数据层
├── domain/               # 领域层
├── presentation/         # 表现层
├── routes/               # 路由配置
└── l10n/                 # 国际化文件
```

### 2. 文件命名
```dart
// 文件命名规范
// ✅ snake_case: home_screen.dart
// ✅ 功能明确: user_model.dart
// ✅ 避免缩写: api_service.dart (不要用 api_svc.dart)
// ✅ 避免特殊字符: 不要用 -, @, # 等

// 目录命名规范
// ✅ snake_case: screens/, widgets/
// ✅ 功能分组: auth/, home/, profile/
// ✅ 层级清晰: presentation/, domain/, data/
```

### 3. 代码组织
```dart
// 代码组织原则
// ✅ 单一职责: 每个文件只做一件事
// ✅ 高内聚低耦合: 相关功能放在一起
// ✅ 依赖倒置: 依赖接口而不是实现
// ✅ 开闭原则: 对扩展开放，对修改关闭
```
