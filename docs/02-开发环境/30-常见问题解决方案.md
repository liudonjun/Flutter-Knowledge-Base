# 开发环境常见问题

> Flutter 开发环境配置中常见问题的解决方案。

## 🔧 环境变量问题

### 1. Flutter 命令找不到

```bash
# 问题：flutter: command not found

# 解决方案 1：检查 PATH
echo $PATH | grep flutter

# 解决方案 2：添加到 PATH
# macOS/Linux
echo 'export PATH="$HOME/flutter/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# Windows
# 系统属性 → 环境变量 → 编辑 PATH → 添加 Flutter 路径

# 解决方案 3：使用绝对路径
~/flutter/bin/flutter --version
```

### 2. Android SDK 找不到

```bash
# 问题：Android SDK not found

# 解决方案 1：设置 ANDROID_HOME
export ANDROID_HOME="$HOME/Library/Android/sdk"
export PATH="$ANDROID_HOME/platform-tools:$PATH"

# 解决方案 2：在 Android Studio 中配置
# File → Settings → Android SDK → SDK Location

# 解决方案 3：使用 flutter config
flutter config --android-sdk /path/to/android/sdk
```

### 3. Java/JDK 问题

```bash
# 问题：JAVA_HOME not set

# macOS
export JAVA_HOME=$(/usr/libexec/java_home)
echo 'export JAVA_HOME=$(/usr/libexec/java_home)' >> ~/.zshrc

# Linux
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64
echo 'export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64' >> ~/.bashrc

# 验证
java -version
echo $JAVA_HOME
```

## 📦 依赖问题

### 1. pub get 失败

```bash
# 问题：pub get failed

# 解决方案 1：清除缓存
flutter pub cache repair

# 解决方案 2：使用镜像
export PUB_HOSTED_URL="https://pub.flutter-io.cn"
export FLUTTER_STORAGE_BASE_URL="https://storage.flutter-io.cn"

# 解决方案 3：检查网络
ping pub.dev

# 解决方案 4：检查 pubspec.yaml 语法
flutter pub deps
```

### 2. 依赖版本冲突

```yaml
# 问题：版本冲突

# 解决方案 1：使用 dependency_overrides
dependency_overrides:
  some_package: ^2.0.0

# 解决方案 2：查看依赖树
flutter pub deps --no-dev

# 解决方案 3：更新所有依赖
flutter pub upgrade --major-versions
```

### 3. CocoaPods 问题

```bash
# 问题：pod install failed

# 解决方案 1：更新 CocoaPods
sudo gem install cocoapods

# 解决方案 2：清除缓存
pod cache clean --all
cd ios && pod deintegrate && pod install

# 解决方案 3：使用国内镜像
cd ios
pod repo add flutter-cocoapods https://mirrors.tuna.tsinghua.edu.cn/git/CocoaPods/Specs.git
pod repo update
```

## 🔨 构建问题

### 1. Android 构建失败

```bash
# 问题：Gradle build failed

# 解决方案 1：更新 Gradle
cd android
./gradlew wrapper --gradle-version=8.0

# 解决方案 2：清除缓存
cd android
./gradlew clean
flutter clean

# 解决方案 3：检查 Java 版本
java -version
# Flutter 需要 Java 11 或更高版本

# 解决方案 4：使用国内镜像
# android/build.gradle
repositories {
    maven { url 'https://maven.aliyun.com/repository/google' }
    maven { url 'https://maven.aliyun.com/repository/public' }
    google()
    mavenCentral()
}
```

### 2. iOS 构建失败

```bash
# 问题：Xcode build failed

# 解决方案 1：更新 Xcode 命令行工具
sudo xcode-select --install

# 解决方案 2：清理项目
cd ios
pod deintegrate
pod install
flutter clean

# 解决方案 3：检查签名
# Xcode → Runner → Signing & Capabilities
# 勾选 "Automatically manage signing"

# 解决方案 4：打开正确的项目
open ios/Runner.xcworkspace  # 不是 .xcodeproj
```

### 3. Web 构建失败

```bash
# 问题：Web build failed

# 解决方案 1：启用 Web 支持
flutter config --enable-web

# 解决方案 2：检查 Chrome
export CHROME_EXECUTABLE=/path/to/chrome

# 解决方案 3：使用不同渲染器
flutter build web --web-renderer html
flutter build web --web-renderer canvaskit

# 解决方案 4：清除缓存
flutter clean
flutter pub get
```

## 🌐 网络问题

### 1. 代理配置

```bash
# 配置 Git 代理
git config --global http.proxy http://proxy.example.com:8080
git config --global https.proxy http://proxy.example.com:8080

# 配置 Flutter 代理
export HTTP_PROXY="http://proxy.example.com:8080"
export HTTPS_PROXY="http://proxy.example.com:8080"

# 配置 Pub 代理
export PUB_HOSTED_URL="https://pub.flutter-io.cn"
export FLUTTER_STORAGE_BASE_URL="https://storage.flutter-io.cn"
```

### 2. 镜像配置

```bash
# 中国镜像配置
export PUB_HOSTED_URL="https://pub.flutter-io.cn"
export FLUTTER_STORAGE_BASE_URL="https://storage.flutter-io.cn"

# 永久配置
echo 'export PUB_HOSTED_URL="https://pub.flutter-io.cn"' >> ~/.zshrc
echo 'export FLUTTER_STORAGE_BASE_URL="https://storage.flutter-io.cn"' >> ~/.zshrc
source ~/.zshrc

# Windows PowerShell
[Environment]::SetEnvironmentVariable("PUB_HOSTED_URL", "https://pub.flutter-io.cn", "User")
[Environment]::SetEnvironmentVariable("FLUTTER_STORAGE_BASE_URL", "https://storage.flutter-io.cn", "User")
```

### 3. DNS 问题

```bash
# 问题：DNS 解析失败

# 解决方案 1：更换 DNS
# macOS
networksetup -setdnsservers Wi-Fi 8.8.8.8 8.8.4.4

# Linux
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf

# 解决方案 2：使用 hosts 文件
# 编辑 /etc/hosts
# 添加 IP 映射
```

## 🔌 设备问题

### 1. 模拟器问题

```bash
# 问题：模拟器无法启动

# 解决方案 1：启用硬件加速
# BIOS 中启用 VT-x/AMD-V

# 解决方案 2：安装 HAXM
# Android Studio → SDK Manager → SDK Tools
# 勾选 Intel x86 Emulator Accelerator

# 解决方案 3：重新创建 AVD
# Android Studio → Device Manager → 删除并重新创建

# 解决方案 4：检查系统要求
# 确保有足够的内存和磁盘空间
```

### 2. 真机调试问题

```bash
# 问题：设备未识别

# Android：
# 1. 启用开发者选项
# 2. 启用 USB 调试
# 3. 信任此电脑
# 4. 检查 USB 连接

# iOS：
# 1. 信任此电脑
# 2. Xcode → Preferences → Accounts → 添加 Apple ID
# 3. 配置签名

# 问题：flutter devices 无输出
flutter doctor -v
# 检查设备连接和配置
```

### 3. 驱动问题

```bash
# 问题：USB 驱动问题

# Windows：
# 1. 安装 OEM USB 驱动
# 2. 设备管理器 → 更新驱动程序

# macOS：
# 通常不需要额外驱动

# Linux：
# 安装 adb
sudo apt install android-tools-adb

# 配置 udev 规则
sudo nano /etc/udev/rules.d/51-android.rules
# 添加：
SUBSYSTEM=="usb", ATTR{idVendor}=="0bb4", MODE="0666", GROUP="plugdev"
sudo udevadm control --reload-rules
```
