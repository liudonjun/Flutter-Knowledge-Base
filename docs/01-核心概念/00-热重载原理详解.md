### 2. 热重载原理

```dart
// 热重载原理
class HotReloadPrinciple {
  void explain() {
    print('''
    热重载原理：
    
    // 1. 代码更新
    // - 检测代码变化
    // - 编译变化的代码
    // - 发送到设备
    // - 替换旧代码
    
    // 2. 状态保持
    // - 保持 Widget 状态
    // - 保持应用状态
    // - 保持导航状态
    // - 保持用户输入
    
    // 3. 更新策略
    // - 选择性更新：只更新变化的代码
    // - 增量更新：只更新改变的文件
    // - 热更新：无需重启应用
    // - 状态重建：重建 Widget 树
    
    // 4. 热重载流程
    // 1. 开发者修改代码
    // 2. IDE 检测变化
    // 3. 编译变化的代码
    // 4. 发送到设备
    // 5. 替换旧代码
    // 6. 重建 Widget 树
    // 7. 保持应用状态
    
    // 5. 热重载示例
    // 修改代码
    class MyWidget extends StatelessWidget {
      @override
      Widget build(BuildContext context) {
        return Text('Hello World');  // 修改这里
      }
    }
    
    // 热重载后
    class MyWidget extends StatelessWidget {
      @override
      Widget build(BuildContext context) {
        return Text('Hello Flutter');  // 更新后
      }
    }
    ''');
  }
}
```
